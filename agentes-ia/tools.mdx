Los Tools amplían las capacidades de un AI Agent al darle acceso a fuentes de información o acciones externas. Usa esta guía para decidir qué herramientas habilitar, cómo configurarlas y cómo combinarlas con tus flujos conversacionales existentes.

## ¿Cuándo usar un Tool?

- **Acceso a datos en tiempo real**: Conecta bases de conocimiento, CRMs o APIs para responder con información actualizada.
- **Automatizar acciones**: Permite que el agente cree tickets, reserve turnos o ejecute workflows cuando el usuario lo solicita.
- **Reducir pasos manuales**: Centraliza interacciones frecuentes para que el agente las resuelva sin intervención humana.

## Proceso recomendado

1. **Define el objetivo**: Identifica la tarea específica que resolverá el Tool y sus límites.
2. **Diseña el prompt**: Incluye instrucciones claras sobre cuándo el agente debe invocar el Tool y cómo interpretar los resultados.
3. **Configura credenciales**: Utiliza variables seguras para claves API u otros secretos necesarios.
4. **Prueba y valida**: Ejecuta escenarios reales con usuarios internos antes de habilitarlo en producción.

## Mejores prácticas

- Mantén los Tools enfocados en una sola responsabilidad.
- Registra y monitorea las ejecuciones para detectar errores o respuestas inesperadas.
- Documenta los parámetros de entrada y salida para que otros equipos puedan reutilizar el Tool.
- Combina múltiples Tools solo cuando el agente pueda decidir claramente cuál usar.

<Tip>
  Comienza con un conjunto reducido de Tools y expándelo a medida que valides casos de uso. Esto te ayudará a mantener el control de calidad y a detectar rápidamente cuando un Tool necesita ajustes.
</Tip>

## Herramientas para Agentes

Descubre cómo conectar herramientas externas y funciones personalizadas con tus agentes para ampliar sus capacidades.

## Buenas prácticas de uso de Tools

- Usa nombres completos y consistentes para las herramientas (sin espacios; reemplázalos por `_` si es necesario).
- Mantén un lenguaje uniforme a lo largo del prompt; evita mezclar términos como “tool”, “función”, “herramienta”.
- Describe con claridad los parámetros que requiere cada tool y su intención.

### Tools nativas

Ejemplo de `send_interactive_message`:

```text
Utiliza la herramienta send_interactive_message con estos parámetros:
{
  caption: <para listas es el nombre del botón>,
  text: <texto principal>,
  title: <texto de arriba del botón/lista>,
  options: [{
    title: <nombre de la opción, obligatorio>,
    description: <descripción, opcional>
  }]
}
```

Notas:
- Con 3 opciones o menos se usa botón.
- Más de 3 opciones usa lista.
- Más de 10, usa texto listado.
